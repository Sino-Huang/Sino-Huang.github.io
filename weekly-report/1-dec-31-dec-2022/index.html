Notes for writing negative paper#

abstract#

  • something to emphasize
    • using language to assist learning agent is a open problem
    • current language reward shaping approach using binary classification proved to have limited capability
    • emphasise that training without language reward is more efficient.
    • the consistently perturbed reward signals is more harmful to the performance.
    • empirical analysis of the tolerance of the model-free reinforcement learning algorithm such as PPO, RND on the perturbed reward
    • Empirical analysis of the tolerance on the compound perturbed rewards that simulate language reward shaping on compound of longer instruction. accumulated false positive reward over a sequence of natural language instruction
  • something to highlight the contribution
    • our focus is the sparse reward setting (which hence require language rewards)
    • we are discussing the tolerance. nost solution.
    • some suggestions about solving the false positive rewards. some suggestions about how to utilise natural language instructions on the learning agent.
    • not noisy setting, but the false positive rewards influence.
  • keyword:
    • false positive rewards,
    • sparse reward setting,
    • tolerance,
    • criticism on language reward shaping,
    • reinforcement learning algorithm
    • Suggestion on the natural language assisted learning. inspire future work on language reward shaping

Future work (for thesis)#

  • the difficulty lies in the hypothetical thinking (e.g., avoid falling off) and conditional action (e.g., wait for the skull to approach you and then jump over it.)
    • In the future, explore the symbolic way to solve the two.
  • Automatically deal with longer instructions; automatically chunk past trajectory into several atomic events
  • how to avoid consistent but imprecise language assisted signals.
    • improve precision -> rephrase the instruction and feed to the model several times and check if the reward is consistent.
    • make it inconsistent -> it has a chance to not give reward signals if the model is not sure.
  • Interpretability -> understand why precision or recall score is low
    • natural language description generation of the past trajectory

Future work (for positive paper)#

  1. remove background
  2. create within level negatives
  3. new project (positive paper)
    1. auto annotation
      1. assume that some players grab the key / or escape the room, they must follow the instruction to some extent, thus we can have the fuzzy match of the human replay and the language instruction pair, and then we can use this as training data
    2. rebuttal the way of calculating language reward -> current method: concatenation of atomic instructions which are manually separated. and use heuristic way to measure
      1. the old way may lead to consistent false positive reward
      2. the old way given Goyal’s experiment, the compound of instructions leads to the product of success rate of each atomic instructions, and thereby causing the final success rate to drop dramatically.
      3. the new way -> description generation based on the past trajectory (random sampling chunks with varying lengths) -> stop until the text generated is cosine similar to the first line of the walkthrough instruction. or if it reaches the maximum memory, we pop out the old memory -> now focus on the second line of the walkthrough instruction and clean the memory and restart text generation, use a new policy network and start the episode from the end of the first trajectory.
      4. no language reward here it is something like a guided exploration
      5. in the end, there is no consistent false positive rewards, the exploration trajectory by the language instruction will be further used as the training replays for the learning agent. and we prioritise the experiences that follows the instruction
